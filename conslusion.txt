1. RESUMEN DE LO APRENDIDO
--------------------------------------------------------------

1.1. Generación de archivos de prueba (GenerateInfoFiles.java):
      - Se aprendió a crear archivos de productos con ID;Nombre;Precio usando
        FileWriter y Random para generar precios aleatorios entre 50 y 550.
      - Se implementó creación de vendedores con tipo de documento (CC),
        ID, nombre y apellido tomados de listas predefinidas.
      - Se desarrolló lógica para crear archivos de ventas por vendedor
        con detección de archivos existentes (si ya existe ventas_1001.txt,
        se crea ventas_1001_2.txt, ventas_1001_3.txt, etc.).
      - Se creó archivo intencionalmente erróneo (ventas_error_demo.txt)
        con ID de producto 99 (inexistente), cantidad negativa (-3) y
        línea con formato inválido para probar validaciones.

1.2. Procesamiento y validación de datos (main.java):
      - Se aprendió a cargar catálogos en memoria usando HashMap:
        productNames (ID -> nombre), productPrices (ID -> precio).
      - Se validaron líneas de archivos con formato incorrecto usando
        split(";") y verificando longitud mínima.
      - Se manejaron excepciones NumberFormatException para líneas con
        valores no numéricos.
      - Se detectaron y reportaron productos con ID inexistente en el catálogo.
      - Se rechazaron cantidades <= 0 con mensajes de advertencia específicos.
      - El programa CONTINÚA procesando líneas válidas aún si encuentra errores.

1.3. Acumulación y generación de reportes:
      - Se acumularon totales por vendedor usando sellerTotals.getOrDefault().
      - Se acumularon cantidades vendidas por producto con productTotals.
      - Se ordenaron listas con Comparator personalizado:
        - Vendedores: de mayor a menor recaudo (Double.compare b vs a).
        - Productos: de mayor a menor cantidad vendida (Integer.compare b vs a).
      - Se generaron archivos CSV con formato compatible con Excel español
        (separador punto y coma).

1.4. Manejo robusto de archivos:
      - Se usó try-with-resources para FileWriter, BufferedReader y FileReader.
      - Se verificó folder.listFiles() != null para evitar NullPointerException.
      - Se filtraron archivos con startsWith("ventas_") AND endsWith(".txt").

1.5. Buenas prácticas de programación:
      - Nombres descriptivos de variables y métodos en camelCase (createSalesMenFile, processSalesFile, generateSellerReport).
      - Constantes implícitas en UPPER_SNAKE_CASE cuando correspondía.
      - Métodos con una sola responsabilidad (cada método hace una cosa específica y bien definida).
      - Documentación Javadoc completa en todas las clases y métodos públicos.
      - Código indentado y espaciado consistentemente para facilitar la lectura.

1.6. Control de versiones con Git:
      - Se inicializó un repositorio local y se conectó a GitHub como repositorio remoto.
      - Se utilizó git pull para mantener el trabajo actualizado con los cambios de los compañeros.
      - Se subieron los cambios al repositorio remoto con git push para tener una copia de respaldo centralizada.



2. POSIBLES APLICACIONES A NIVEL PROFESIONAL
--------------------------------------------

2.1. Sistemas de facturación y ventas:
      - Procesamiento de archivos de ventas diarios desde múltiples puntos de venta.
      - Validación automática de que los productos existan en el catálogo central.
      - Generación de reportes de comisiones para vendedores.

2.2. Control de inventarios:
      - El acumulador productTotals permite saber qué productos se venden más.
      - Se puede extender para alertar cuando un producto tiene baja rotación.

2.3. Pruebas de software (QA):
      - GenerateInfoFiles puede generar datos masivos para probar sistemas.
      - El archivo ventas_error_demo.txt sirve para probar manejo de errores.

2.4. Migración de datos:
      - Empresas que reciben archivos planos de proveedores pueden usar
        este patrón para validar y procesar la información antes de cargarla.

2.5. Base para proyectos más grandes:
      - Cambiando FileReader por JDBC se conecta a base de datos.
      - Agregando interfaz gráfica (Swing/JavaFX) se hace amigable al usuario.
      - Exponiendo como API REST con Spring Boot se convierte en microservicio.

2.6. Trabajo colaborativo con Git en empresas:
      - El flujo de trabajo con ramas, commits semánticos y pull requests es exactamente el mismo que se usa en equipos de desarrollo 	profesional.
